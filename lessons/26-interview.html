3-qism

Tez-tez so'raladigan savollar

Intervyuda albatta so'raladigan 15 ta savol va qisqa pro javoblar. Bosib ko'ring.

Bug va Error farqi nima? +
Bug — dastur kutilganidan boshqa ishlayotganda (test topadi). Error — ishga tushirishni to'sqinlovchi xato (sintaksis, runtime). Bug — funksional muammo, Error — texnik muammo.
Smoke va Sanity testing farqi? +
Smoke — yangi build'ning asosiy funksiyalari ishlayaptimi? (Keng, sayoz). Sanity — bug fix'dan keyin faqat o'sha modul to'g'rilandimikin? (Tor, chuqur).
Severity va Priority farqi? +
Severity — bug dasturga qanchalik zarar yetkazadi (texnik ta'sir). Priority — tuzatish qanchalik tez kerak (biznes muhimligi). Misol: Sarlavhadagi imlo xatosi — Severity Low, lekin Priority High (bosh sahifada ko'rinadi).
Black box va White box testing? +
Black box — ichki kodni bilmasdan test qilish. Foydalanuvchi ko'zi bilan. QA asosan shu. White box — kod ko'rinadi, ichki mantiqni test qilish. Dasturchi qiladi.
Regression testing nima? +
Yangi o'zgartirish eski funksiyalarni buzmadimi? — tekshirish. Har yangi release'dan oldin eski funksiyalar qayta test qilinadi. Automation uchun ideal.
Agile va Scrum nima? +
Agile — iterativ rivojlantirish metodologiyasi. Qisqa tsikllar, moslashuvchanlik. Scrum — Agile'ning bir framework'i. Sprint (1-2 hafta), Daily standup, Sprint review. QA Sprint davomida test qiladi.
SDLC va STLC nima? +
SDLC — Software Development Life Cycle: Talablar → Dizayn → Rivojlantirish → Test → Deploy → Qo'llab-quvvatlash. STLC — Software Testing Life Cycle: Talablarni tahlil → Test rejasi → Test case → Test muhit → Test bajarish → Test yopish.
POM nima va nima uchun kerak? +
Page Object Model — har sahifa uchun alohida class. Locatorlar va metodlar shu classda. Afzalligi: locator o'zgansa — 1 joyda o'zgartirish yetarli, test kodi toza va tushunarli.
GET va POST farqi? +
GET — ma'lumot olish, URL'da ko'rinadi, xavfsiz. POST — ma'lumot yaratish, body'da yuboriladi, URL'da ko'rinmaydi. GET — o'qish, POST — yozish.
p95 nimani anglatadi? +
p95 = 1.8s degani: 100 ta so'rovdan 95 tasi 1.8 sekunddan tez bajarildi. Faqat 5 tasi sekinroq. O'rtacha aldashi mumkin — p95/p99 haqiqiy holat.
Test Pyramid nima? +
Automation strategiyasi: Unit 70% (tez, ko'p), API 20% (o'rta), UI/E2E 10% (sekin, kam). Ko'p tez testlar + oz sekin testlar = ishonchli va samarali.
Boundary Value Analysis nima? +
Chegaraviy qiymatlarni test qilish. Misol: yosh 18-65 oralig'ida qabul qilinadi → 17, 18, 65, 66 test qilinadi. Chegarada ko'proq bug bo'ladi.
Exploratory testing nima? +
Oldindan yozilgan test case'siz, erkin tekshirish. QA o'zi fikrlaydi, sinaydi, yangi yo'llar izlaydi. Yangi funksiya yoki vaqt kam bo'lganda yaxshi. Tajriba va intuitsiya muhim.
Defect Life Cycle nima? +
Bug'ning hayot tsikli: New → Assigned → Open → Fixed → Retest → Closed (yoki Reopened). QA New qiladi, dasturchi Fix qiladi, QA Retest qiladi va Close yoki Reopen qiladi.
Equivalent Partitioning nima? +
Kiritish qiymatlarini guruhlarga bo'lish — har guruhdan 1 ta test yetarli. Yosh field: 0-17 (yaroqsiz), 18-65 (yaroqli), 66+ (yaroqsiz). Har guruhdan 1 ta test qilish — hammani tekshirgandek.
