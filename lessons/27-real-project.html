2-qism

QA'ning kundalik ishi

Agile muhitida oddiy ish kuni qanday o'tadi?

Kundalik routine (Agile/Scrum)
  1. 09:00 — Daily Standup (15 daqiqa): Kecha nima qildim, bugun nima qilaman, to'siq bormi?
  2. 09:15 — Test caselar yozish: Yangi feature uchun test caselar tayyorlash
  3. 10:30 — Test bajarish: Yangi build'ni test qilish, buglarni topish
  4. 12:00 — Tushlik 🍽️
  5. 13:00 — Bug hujjatlash: Topilgan buglarni Jira'da yozish, dasturchiga assign
  6. 14:00 — Retest: "Fixed" deb belgilangan buglarni qayta tekshirish
  7. 15:00 — Automation yoki API test: Agar sprint'da bor bo'lsa
  8. 17:00 — Status update: Jira'da vazifalar holati yangilash

Jamoa bilan muloqot

  • Dasturchiga bug topganingizda: "X sahifada Y qilganda Z xatosi chiqadi, Jira'da #456 — tekshirib ko'ra olasizmi?"
  • Talabni tushunmasangiz: "Bu funksiya qanday ishlashi kerak? Test qilishim uchun aniqlashtirish mumkinmi?"
  • Bug muhimmi-yo'qmi: "Bu Critical deb belgilasam to'g'rimi, yoki biznes uchun qanchalik muhim?"
  • Yordam kerak bo'lsa: "Bu texnik momentni tushunishimda yordam bera olasizmi?"
Dasturchiga xabar yozish qoidasi

Har doim: Bug ID + qisqa tavsif + qadam-baqadam. "Login ishlamaydi" emas — "Jira #456: Login'da to'g'ri parol bilan 500 xato. Steps: email ⟶ parol ⟶ kirish. Chrome 120."

💡 Har savol ustiga bosing — professional javobni ko'rasiz.

Q1Birinchi ish kuni nima qilasiz?+

Pro javob:

  • Jamoani tanishaman — QA Lead, dasturchilar, PM
  • Mahsulotni foydalanuvchi sifatida o'rganaman
  • Asboblar: Jira, TestRail, Slack, Git — kiraman va o'rganaman
  • Eski bug reportlar va test caselarni o'qiyman
  • Savollar ro'yxati yozaman — hamma narsani eslab qolish kerak emas

Muhim: "Ko'raman va tushunaman" emas — "Ko'raman, savol beraman, yozaman."

Q2Jamoada qanday ishlaysiz?+

Pro javob:

  • Daily standup'da kecha/bugun/to'siq — aniq va qisqa
  • Bug topganda dasturchiga Jira ID + qisqa tavsif bilan xabar
  • Talablar noaniq bo'lsa — PO'dan so'rayman, taxmin qilmayman
  • Release qaroriga hissa — "Bu risk bor" deb aytaman
  • Jamoa maqsadi uchun — o'z ishi bilan birga boshqalarga ham yordam
Q3Deadline bosimida qanday qaror qabul qilasiz?+

Pro javob:

"Deadline bosimida eng muhim qaror — halol bo'lish."

  • Nima test qilingan, nima qilinmagan — aniq aytaman
  • Risk baholash: "Bu qismni skip qilsak qanday xavf bor?"
  • Qaror PMda — men ma'lumot beraman, qaror qabul qilmay
  • "Hammasi yaxshi" deyish — bu yolg'on va xavfli

Formula: "Nima qilingan + nima qilinmagan + xavf = PM qaror qabul qiladi."

Q4Dasturchiga bug qanday etkaziladi?+

Pro javob:

  • Jira'da to'liq yozaman: Title, Steps, Expected, Actual, Screenshot, Severity
  • Slack'da Jira ID bilan xabar: "Hey, #456 — Login'da Critical bug, tekshira olasizmi?"
  • Conflict emas — hamkorlik: "Bu bug haqida gaplashamizmi?"
  • Fix qilingan deyilganda — doim Retest qilaman, yopmayman

Muhim: Bug topish — dasturchiga hujum emas. Mahsulot uchun birgalikda ishlash.

Q53 oy ichida nima o'rganmoqchisiz?+

Pro javob:

  • 1-oy: Mahsulotni tushunish, jarayon va asboblarni o'rganish, mustaqil test caselar
  • 2-oy: Release tsiklida to'liq ishtiroki, API testing kuchaytirish
  • 3-oy: Automation boshlash, birinchi Playwright skriptlar, CI/CD bilan tanishish

Bu — recruiter uchun: "Siz faqat ish emas, o'sishni ham o'ylayapsiz" taassurotini qoldiradi.

Q6Junior va Middle QA farqi nima?+

Junior QA:

  • Berilgan vazifalarni bajaradi
  • Test caselar va bug reportlar yozadi
  • QA Lead rahbarligida ishlaydi
  • Manual testing asosiy

Middle QA:

  • Mustaqil qarorlar qabul qiladi
  • Automation framework yaratadi
  • Test strategiyasini belgilaydi
  • Junior'larga mentor bo'ladi
  • API, performance, mobile — barcha yo'nalishlar

O'tish: 1-2 yil ish + automation bilimi + leadership ko'nikmalari.
