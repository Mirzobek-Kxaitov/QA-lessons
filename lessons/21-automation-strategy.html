1-qism

Page Object Model (POM)

POM — eng mashhur automation pattern. Google'dan tortib barcha katta kompaniyalar foydalanadi.

POM nima?

Page Object Model — har bir sahifa uchun alohida Python class. Sayt elementlari (locatorlar) va ular bilan ishlash metodlari shu classdа yig'iladi.

Oddiy qilib: LoginPage degan class — login sahifasining hamma narsasini biladi. Test fayllar bu classni ishlatadi, lekin locatorlarni bilmaydi.

POM'siz (muammo)

❌ Yomon # test_login.py driver.find_element(By.ID, "email").send_keys("test@test.com") driver.find_element(By.ID, "password").send_keys("123") driver.find_element(By.ID, "login-btn").click() # test_checkout.py — xuddi shu locatorlar qayta! driver.find_element(By.ID, "email").send_keys("buyer@test.com") driver.find_element(By.ID, "password").send_keys("456") driver.find_element(By.ID, "login-btn").click() # "login-btn" → "btn-login" o'zgarse: 20 ta faylda o'zgartiramiz!

POM bilan (yechim)

✅ Yaxshi # pages/login_page.py — LoginPage class class LoginPage: def __init__(self, page): self.page = page def login(self, email, password): self.page.fill("#email", email) self.page.fill("#password", password) self.page.click("#login-btn") # test_login.py — faqat 1 qator! LoginPage(page).login("test@test.com", "123") # "login-btn" o'zgarse: FAQAT login_page.py da o'zgartiramiz!

POM afzalliklari

  • O'zgartirish oson — locator 1 joyda, 50 ta testni emas
  • O'qish oson — LoginPage.login() — nima qilayotgani aniq
  • Qayta ishlatish — login logikasi har qaysi testda tayyor
  • Jamoa bilan ishlash — har developer o'z sahifasini yozadi
Asosiy qoida

1 sahifa = 1 class. LoginPage, ProductPage, CartPage, CheckoutPage. Test fayllar faqat bu classlarni chaqiradi, locatorlarni bilmaydi.

🔄 Before / After

POM'siz vs POM bilan — farqni ko'ring

Bir xil vazifa — 2 xil yondashuv. Qaysi biri yaxshiroq ekanini o'zingiz baholang:

❌ POM'siz (10 ta test faylda)
driver.find_element( By.ID, "login-btn" ).click()
10 ta faylda xuddi shu kod takrorlanmoqda. "login-btn" o'zgarse — 10 ta faylda qidirasan!
✅ POM bilan (1 joyda)
LoginPage(page).login( email, password )
"login-btn" o'zgarse — faqat LoginPage classida 1 qator o'zgartirish. Hammasi tayyor!
❌ POM'siz (test faylda)
driver.find_element( By.CLASS_NAME, "add-cart" ).click() driver.find_element( By.ID, "cart-count" ).text
Locatorlar test faylida. Sayt o'zgarsa — hamma test fayllarni tekshirish kerak.
✅ POM bilan
product = ProductPage(page) product.add_to_cart() count = product.get_cart_count()
Locatorlar ProductPage classida. Test kodi toza va tushunarli.
❌ POM'siz (5 oy keyin)
# Bu nima qilyapti? driver.find_element( By.XPATH, "//div[@class='hdr']//a[2]" ).click()
XPath nima ekanini tushunish qiyin. 5 oy keyin o'zing ham unutasan!
✅ POM bilan
nav = NavPage(page) nav.go_to_profile()
Metod nomi — nima qilayotgani aniq. O'qish oson, tushunish oson.
2-qism

Pytest — testlarni ishlatish

Testlarni qo'lda ishlatish — python test_login.py. Lekin 50 ta test bo'lsa? Pytest — hammasini bir buyruqda ishlatadi.

Pytest nima?

Pytest — Python'da eng mashhur test framework. Testlarni topadi, ishlatadi, natijani chiroyli ko'rsatadi.

O'rnatish: pip install pytest pytest-playwright

Pytest qoidalari (3 ta)

test_rules.py # Qoida 1: Fayl nomi test_ bilan boshlanishi kerak # test_login.py ✅ — login.py ❌ # Qoida 2: Funksiya nomi test_ bilan boshlanishi kerak def test_login_success(): # ✅ ... def check_login(): # ❌ pytest topa olmaydi ... # Qoida 3: assert bilan tekshirish assert page.title() == "TestShop" # ✅ assert page.is_visible("text=Welcome") # ✅

Birinchi Pytest testi

test_login.py from playwright.sync_api import Page def test_login_success(page: Page): page.goto("https://testshop.uz/login") page.fill("#email", "test@uzum.uz") page.fill("#password", "Pass123!") page.click("#login-btn") assert page.is_visible("text=Welcome") def test_login_wrong_password(page: Page): page.goto("https://testshop.uz/login") page.fill("#email", "test@uzum.uz") page.fill("#password", "XATO123") page.click("#login-btn") assert page.is_visible("text=Noto'g'ri parol")

Ishlatish — terminal

terminal # Barcha testlarni ishlatish pytest # Bitta faylni pytest test_login.py # Natijani batafsil ko'rish pytest -v # Playwright bilan (brauzer ko'rinadigan) pytest --headed

Fixture — brauzer setup

Fixture — har test oldidan bajariladigan tayyorgarlik. Masalan: brauzer ochish, login qilish.

conftest.py import pytest from playwright.sync_api import sync_playwright # Bu fixture har testdan oldin brauzer ochadi @pytest.fixture def page(): with sync_playwright() as p: browser = p.chromium.launch(headless=False) page = browser.new_page() yield page # Test shu yerda ishlaydi browser.close() # Test tugagach yopiladi
Sodda qoida

test_ bilan boshla (fayl va funksiya). assert bilan tekshir. pytest bilan ishga tushir. Shu 3 ta narsa yetarli!

Q1Page Object Model nima va nima uchun kerak?+

Javob: "POM — har sahifa uchun alohida Python class. Sahifaning locatorlari va metodlari shu classda to'planadi."

Nima uchun kerak:

  • O'zgartirish oson — locator 1 joyda, 50 ta testda emas
  • Qayta ishlatish — LoginPage.login() har yerda
  • O'qish oson — test kodi toza

Pro javob: "Amalda ishlatganman. Testshop loyihasida LoginPage, ProductPage, CartPage class yaratdim. Locator o'zganda — faqat 1 faylda o'zgartirdim."

Q2Pytest nima va qanday ishlatiladi?+

Javob: "Pytest — Python'da eng mashhur test framework. Test fayllarni topadi, ishlatadi, natijani ko'rsatadi."

Qoidalar:

  • Fayl: test_*.py yoki *_test.py
  • Funksiya: test_ bilan boshlanadi
  • Tekshirish: assert bilan

Buyruqlar: pytest — hammasi, pytest -v — batafsil, pytest --headed — Playwright bilan brauzer ko'rinadigan.

Q3Fixture nima?+

Javob: "Fixture — har testdan oldin bajariladigan tayyorgarlik. Masalan, brauzer ochish, test bazasiga ulanish, login qilish."

Misol: conftest.py'da @pytest.fixture def page(): — har test funksiyasi page parametrini oladi, brauzer tayyor holda keladi.

Afzalligi: Har test faylida brauzer ochish kodi yozilmaydi — 1 joyda yoziladi, hammaga ishlaydi.

Q4Automation loyiha strukturasi qanday bo'lishi kerak?+

Javob:

  • pages/ — Page Object classlar (login_page.py, product_page.py)
  • tests/ — test fayllar (test_login.py, test_checkout.py)
  • conftest.py — fixture'lar (brauzer setup)
  • requirements.txt — kutubxonalar ro'yxati
  • pytest.ini — Pytest sozlamalari

Pro javob: "Men shu strukturani ishlataman. Yangi developer loyihaga kirsa — 5 daqiqada tushunadi."

Q5Test muvaffaqiyatsiz bo'lganda nima qilasiz?+

Javob: "Test xato berganda ketma-ket tekshiraman:"

  • Screenshot — Playwright'da page.screenshot() — test paytida nima ko'ringanini tekshiraman
  • Xato matni — AssertionError qaysi qatorda ekani
  • Locator — element hali ham sahifada bormi? ID o'zgarganmi?
  • Wait — element yuklanishini kutish yetarlimikin?
  • Playwright Trace — qadam-baqadam videoni ko'rish
Q6Automation tajribangizni aytib bering+

Pro javob (to'liq):

"Men Python + Playwright + Pytest bilan ishlashni o'rgandim. TestShop loyihasida automation framework yaratdim:"

  • Framework: Playwright (Python), Pytest
  • Pattern: Page Object Model — LoginPage, ProductPage, CartPage
  • Struktura: pages/, tests/, conftest.py
  • Testlar: Login, mahsulot ko'rish, savatga qo'shish
  • GitHub: Loyiha GitHub'da, README bilan

Bu — mening birinchi POM loyiham. Hozir Selenium bilan ham ishlashni bilaman, yangi loyihada Playwright tavsiya qilaman.

Q7POM'siz testlar bilan POM'li testlar farqi?+

Javob: "Amaliy farq — locator o'zganda:"

  • POM'siz: #login-btn 50 ta faylda bor. O'zgansa — 50 ta faylda qidirish. 1-2 soat ish.
  • POM bilan: #login-btn faqat LoginPage classida. O'zgansa — 1 qator. 1 daqiqa.

Ko'rinadigan farq (test fayl):

  • POM'siz: driver.find_element(By.ID, "login-btn").click()
  • POM bilan: LoginPage(page).login(email, password)

POM bilan test kodi nima qilayotganini ko'rsatadi, qanday qilayotganini emas.
