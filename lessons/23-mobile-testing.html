Nima uchun Mobile Testing muhim?

Uzum, Payme, Click — bu ilovalarni foydalanuvchilarning 80%+ mobil telefonda ishlatadi. Siz QA sifatida faqat web saytni emas, mobil ilovani ham sinab ko'rishingiz kerak.

💡 Sodda misol
Bir xil do'kon — ikki xil eshik
Web sayt — asosiy eshik. Mobil ilova — yon eshik. Ikkalasini ham tekshirish kerak! Asosiy eshik yaxshi bo'lsa ham, yon eshik buziq bo'lishi mumkin.

Mobil va Web testing farqi

  • Ekran o'lchami — 5" telefon, 10" planshet, 27" monitor — hammasi boshqacha
  • Sensorli ekran — sichqon yo'q, barmaq bilan ishlash
  • Tarmoq — Wi-Fi, 4G, 3G, hatto 2G da ishlashi kerak
  • Batareya — ilova batareyani ko'p sarflayaptimi?
  • Notifikatsiya — keldi, ilova qanday reaksiya qiladi?
  • Ko'p qurilma — Samsung, Xiaomi, iPhone, iPad — hammasi boshqacha
Bozor ulushi (2024)

Android — 72% global bozor. iOS — 27%. O'zbekistonda Android ancha ko'p (arzonroq telefonlar). Ikkalasi ham muhim!

Muhim: Interrupt Testing

Bu Web testingda yo'q, faqat mobilda bor! Ilova ishlayotganda:

Interrupt Test scenariylari
  • Qo'ng'iroq keldi → Ilova to'xtatildi → Qo'ng'iroq tugatildi → Ilova davom etdi. Ma'lumot yo'qolmadimikin?
  • SMS yoki bildirishnoma keldi → Ekran yuqorida banner ko'rsatdi → Banner bosildi → Ilova qanday reaksiya berdi?
  • Ilova foniga tushdi (Home tugma) → Qaytib keldim → Holat saqlandimikin?
  • Batareya qizidi → Tizim ilovani chekladi → Ilova xato bermadimi?

Mobile testing asboblari

Manual testdan tashqari, mobil ilovalarni tekshirishga yordam beradigan asboblar.

Android Developer Options

Android qurilmada yashirin sozlamalar bor — QA uchun juda foydali:

  • Yoqish: Sozlamalar → Telefon haqida → "Build number"'ni 7 marta bosing
  • USB Debugging — kompyuter bilan ulash, ADB buyruqlari
  • Show Touches — barmoq tegishlarini ko'rsatadi (video uchun)
  • GPU Rendering — animatsiya muammolarini ko'rish
  • Background process limit — kam RAM sharoitini simulyatsiya

Android Studio Emulator

Android Studio Emulator

Bepul. Google tomonidan. Mac, Windows, Linux. Turli Android versiya va qurilmalarni simulyatsiya qiladi.

Afzalligi: Ko'p qurilma o'rnatmasdan turli ekran va Android versiyalarini sinash mumkin. Tarmoq tezligini ham kamaytirish mumkin (2G, 3G simulyatsiya).

iOS Simulator (Xcode)

Xcode Simulator

Bepul, faqat Mac'da. Apple tomonidan. iPhone va iPad turlarini simulyatsiya qiladi.

Muhim: Simulator — haqiqiy telefon emas. GPS, kamera, push notification — cheklangan. Release oldidan real iPhone kerak.

BrowserStack va Sauce Labs

Agar ko'p real qurilmada test kerak bo'lsa — bulutdagi qurilmalar:

  • BrowserStack — 3000+ real qurilma bulutda. Pullik (bepul trial bor)
  • Sauce Labs — shunga o'xshash, enterprise uchun
  • Firebase Test Lab — Google'ning bepul testi (cheklangan)

ADB — Android Debug Bridge

Terminal orqali Android qurilmani boshqarish:

# Ulangan qurilmalar ro'yxati
adb devices

# Screenshot olish
adb shell screencap /sdcard/screen.png

# APK o'rnatish
adb install app.apk

# Log ko'rish (crash topsangiz)
adb logcat

Intervyuga tayyorlov

Intervyu savollari

💡 Har savol ustiga bosing — professional javobni ko'rasiz.

Q1Mobile va Web testing farqi qanday?+

Javob: "Mobile testingda web'da bo'lmagan qo'shimcha sohalar bor:"

  • Qurilma fragmentatsiyasi — yuzlab telefon turi, har xil ekran, Android versiya
  • Interrupt testing — qo'ng'iroq, SMS, bildirishnoma — ilovaga ta'siri
  • Sensorlar — GPS, kamera, akselerometr, barmoq izi
  • Tarmoq o'tish — Wi-Fi → 4G → 3G → tarmoqsiz
  • Batareya va resurslar — ilova RAM va batareyani nechta iste'mol qiladi
  • Ekran yo'nalishi — Portrait/Landscape
Q2Android va iOS testingda qanday farqlar bor?+
  • Fragmentatsiya: Android'da 1000+ qurilma, iOS'da faqat Apple qurilmalar
  • Emulator/Simulator: Android Studio (Windows/Mac/Linux), Xcode Simulator (faqat Mac)
  • O'rnatish: Android — APK to'g'ridan-to'g'ri, iOS — TestFlight yoki App Store
  • Test osonligi: iOS qurilmalar oz → test osonroq. Android ko'p → ko'p tekshiruv
  • Back tugmasi: Android'da fizik yoki virtual "orqaga" tugma — iOS'da yo'q
Q3Real qurilma va emulator qachon ishlatiladi?+

Emulator/Simulator:

  • Dastlabki va doimiy testlar — tez va arzon
  • Ko'p versiya va ekran o'lchamini parallel test qilish
  • CI/CD pipeline'da automation

Real qurilma:

  • GPS, kamera, barmoq izi kabi sensor testlar
  • Haqiqiy performance (batareya, tezlik)
  • Release oldidan final acceptance test
  • Foydalanuvchi tajribasini to'liq tekshirish

Pro javob: "Ikkala yondashuv birgalikda ishlatiladi. Emulator tez va arzon, real qurilma ishonchli."

Q4Interrupt testing nima va misollar?+

Javob: "Interrupt testing — ilova ishlayotganda tashqi voqea bo'lganda qanday reaksiya berishi."

Scenariylar:

  • Qo'ng'iroq keldi → ilova to'xtatildi → tugatildi → davom etdi
  • SMS/bildirishnoma keldi → banner bosib ko'rish → ilova holati
  • Ilova background'ga tushdi va uzoq kutib turdi → session muddati
  • Batareya qizidi → tizim resurslarni chekladi → ilova ishlayaptimi
  • Ekran o'chdi → yondi → ilova holati saqlandimikin

Bu nima uchun muhim: Real foydalanuvchilar doim faqat bir ilovada bo'lmaydi. Bu realistik holatlar.

Q5Mobile bug topildi — qanday hujjatlashtirmasiz?+

Javob: "Mobile bug hujjatida qo'shimcha ma'lumotlar bo'ladi:"

  • Qurilma: Samsung Galaxy S23, Android 13
  • Ilova versiyasi: v2.1.4 (build 124)
  • Tarmoq: Wi-Fi / 4G / Offline
  • Qadam-baqadam: Portrait rejimda login → ...
  • Screenshot yoki video — mobil bug uchun video juda muhim
  • Qayta takrorlash: Faqat bu qurilmadami yoki boshqalarda ham?

Pro maslahat: ADB bilan adb logcat — crash log'ini ham qo'shing. Dasturchiga juda yordam qiladi.

Q6Mobile testing tajribangiz bormi?+

Pro javob:

"Ha, Android va iOS ilova testini o'tkazganman. TestShop loyihasida:"

  • Android Studio Emulator — dastlabki funksional testlar
  • Real Samsung qurilma — yakuniy test, interrupt testing
  • iOS Simulator (Mac) — asosiy UI tekshiruv
  • Checklist: Portrait/Landscape, tarmoq o'tish, bildirishnoma, yangilash
  • Bug hujjat: Qurilma nomi, OS versiya, video bilan

Bu — Junior darajasida zo'r tajriba. Emulator va real qurilmada ikkalasida ham ishlaganman.

Q7Appium nima va nima uchun kerak?+

Javob: "Appium — mobil ilovalar uchun automation testing framework. Selenium'ning mobil versiyasi desak ham bo'ladi."

  • Android ham iOS — bitta framework bilan ikkalasi
  • Python, Java, JS — tanlagan tilda yozish mumkin
  • Native, Hybrid, Web — har xil ilova turini qo'llab-quvvatlaydi

Junior uchun: Hozircha manual mobile testing yetarli. Appium — Middle/Senior QA uchun. Lekin nima ekanini bilish intervyuda foydali.

Pro javob: "Appium haqida bilaman — mobile automation uchun. Hozircha manual testga e'tiborim ko'p, keyingi qadam Appium o'rganish."
